Last Edit Time Feature - Browser Test

Instructions:

  1. IMPORTANT: First, hard refresh your browser: Ctrl + Shift + R
  2. Open your application in the browser
  3. Go to Condition Database
  4. Perform the tests below

Test 1: Manual Edit Creates Timestamp

Action: Edit any condition and save changes

Expected: Shows "Last edited on: [current date/time]" with green "Permanent ✓" badge

Status: ⏳ Pending

Test 2: Export Includes lastEditTime

Action: Click "Export" button in Condition Database

Expected: Downloaded JSON file contains "lastEditTime" field

Check: Open the JSON file and search for "lastEditTime"

Status: ⏳ Pending

Test 3: Import Preserves Original Timestamp

Action: Import the JSON file you just exported

Expected: Shows ORIGINAL date (NOT today's date)

Example: If you edited at 2:30 PM and import at 3:00 PM, should still show 2:30 PM

Status: ⏳ Pending

Test 4: Import Without Timestamp Shows Nothing

Action: Create/import a JSON file WITHOUT lastEditTime field

[
  {
    "id": "test-1",
    "name": "Test Condition",
    "symptoms": ["fever", "headache"]
    // NO lastEditTime field
  }
]

Expected: Shows NOTHING for "Last edited", appears at BOTTOM of list

Status: ⏳ Pending

Test 5: Sorting Works Correctly

Expected Order:

  1. Conditions WITH lastEditTime (sorted newest first)
  2. Conditions WITHOUT lastEditTime (at the bottom)

Status: ⏳ Pending

Quick Verification Checklist:

  • Hard refreshed browser (Ctrl + Shift + R)
  • Edited condition shows timestamp
  • Export includes lastEditTime field
  • Import preserves original date (not import time)
  • Conditions without timestamp at bottom
  • No auto-generation of timestamps

Troubleshooting:

If tests fail:

  1. Clear browser cache completely
  2. Hard refresh: Ctrl + Shift + R
  3. Close and reopen browser
  4. Check browser console for errors (F12)
  5. Verify you're running latest build
